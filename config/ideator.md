# 起案ブリーフ（未設定）

このファイルは `/loop-setup` で利用目的に合わせて構築する。
`ideator`（起案者）がこの用途で「案」を**どう構成し・どんなスタンスで発想するか**を定義するインスタンス設定。
（critic が `config/axes/`、gatekeeper が `config/gates/` を読むのと同じく、ideator はこのファイルを読む。）

ここで定義するもの：
- **出力フォーマット**：案に必ず含める見出し。`config/gates/` と `config/axes/` が問う観点を確実にカバーできる構成にする。
  - 例：事業創出＝タイトル／一言サマリ／顧客と課題／解決策／収益モデル／模倣困難性／スケール。
  - 例：組織改善＝課題／改善案／コスト対効果／実現手順／リスク・整合。
- **起案スタンス（任意）**：この用途固有の発想方針（例：新規性より再解釈・転用を重視／トレンドの扱い 等）。

> 前提・強み・制約・除外は `config/founder-profile.md` 側に書く（ここは「案の作り方」だけ。ダブらせない）。

まだ設定されていません。`/loop-setup` を実行してください。
