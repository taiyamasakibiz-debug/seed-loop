# CLAUDE.md — SEED-LOOP

このプロジェクトの常時コンテキスト。詳細は [README.md](./README.md)、ループ設定は [LOOP.md](./LOOP.md)、状態は [STATE.md](./STATE.md)。

## これは何か
「**起票 → 足切り → 審査 → 人間承認**」を回す汎用の自己改善ループ。
seed（課題・テーマ）を入れると AI が案を起票し、独立した審査 AI が厳しく採点し、合格案だけを人間が最終承認する。
用途は `config/` 次第（例：ビジネスアイデア創出／既存業務の改善提案 など）。設定は `/loop-setup` で構築する。

## 構成の原則（エンジン / インスタンス分離）
- **エンジン（`.claude/`）** = 汎用・ドメイン非依存。`ideator` / `gatekeeper` / `critic` と `run-loop`。
- **インスタンス（`config/`）** = 今回の設定。差し替え可能。
- `critic` は `config/axes/` のファイル数だけ、`gatekeeper` は `config/gates/` の基準数だけ評価する。
  → **評価軸・足切りの増減は config のファイル追加/削除のみ。エンジンは書き換えない。**

## ループの回し方
- `/run-loop [seed]` で1サイクル。
- 合格 = **全採点軸 80点以上の AND**。足切り = `config/gates/`。修正 = 最大3回 ＋ 停滞検知。

## 検証の前提
- **すべての判定は `config/founder-profile.md`（前提プロファイル）に従う。** 一般論で判定しない。
- 採点は**反証ファースト**（まず欠陥を3つ → 減点根拠つきで採点）。甘い満点を出さない。各 axis には**点数バンド**（合格80の意味）を必ず持たせる。
- 軸（axes）と足切り（gates）の中身は `config/` で定義する。利用目的ごとに `/loop-setup` で作る。

## ファイル規約
- 承認待ち: `ideas/pending/*.md`（frontmatter `status: pending`）
- 承認: 人間が `status` を `approved` / `rejected` に書き換える（`rejected` はコメント必須、最優先で修正へ）
- 承認済み: `ideas/approved/`
- 没: `ideas/boneyard/*.md`（`killed_by` と没理由を必ず記録。seed 昇格判断に使う）
- 種: `config/seeds/`（関心領域・課題・キーワード。没ネタから人間が手動昇格する）

## やらないこと（安全方針）
- **Web からのプロンプト自動改変はしない**（自己進化の自動化は廃止）。
- **GitHub 等の自律クローン＆実行はしない**。人間が検証・承認したものだけ取り込む。
- 汎用タスクの Skill 化・思考のチューニングは、**人間の承認を経て**行う。

## 作業の作法
- 設定（config）や成果物（ideas）の変更はこまめにコミットする。
- 破壊的・外向きの操作は事前に確認する。
