# Gate: <ゲート名>（足切り）

> これは**足切りゲート**。`gatekeeper` が採点（critic）の**前**に評価する。出力は二値（PASS / FAIL）。
> FAIL したアイデアはその時点で打ち切り、boneyard に理由付きで記録される。
> 前提は `config/founder-profile.md` を参照すること。
> ※ `config/gates/` に複数のゲートを置けば、gatekeeper はそれぞれを評価する（どれか1つでも FAIL なら全体 FAIL）。

## 判定する問い（二値）
**「<このゲートが守る一線を、Yes/Noで答えられる問いとして>」**
（例：この企画は、本人の制約内で“ほぼ初期投資ゼロ”で立ち上げられるか？）

## PASS 条件（すべて満たすこと）
- <条件1>
- <条件2>
- <条件3>

## HARD FAIL（1つでも該当したら即 FAIL）
- <即不合格の条件1>
- <即不合格の条件2>

## 例外・注意
- <例外規定。後フェーズで重くなるものは別軸(例: finance)に回す等の切り分け>
- **迷ったとき**：守るべき希少資源（資金・時間など）を守る方向に倒す（疑わしきは FAIL 寄り）。

## 出力フォーマット（gatekeeper）
```
判定: PASS | FAIL
理由: （FAILなら該当条項を具体的に。PASSなら一言で根拠）
```
